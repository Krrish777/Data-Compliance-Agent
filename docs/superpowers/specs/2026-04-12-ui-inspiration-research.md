# UI Design Inspiration Research — AuditLens

> Key findings from compliance tools (Vanta, Secureframe, SecurityScorecard), editorial web design (Stripe Annual Letter, ProPublica, FT, GOV.UK), and anti-pattern analysis.

## Top Techniques to Steal

### From Stripe Annual Letter
- Fluid type: `calc()` based on viewport width (document scales like print)
- `text-wrap: balance` on headings (prevents widows — print convention)
- `letter-spacing: -0.96px` on large headings (optical tightening)
- Skewed section dividers: `transform: skewY(-6deg)`
- Content column 1080px max — feels like a page, not a screen

### From ProPublica
- Two optical sizes of same typeface family (Text for small, Headline for large)
- 50-70 character line length enforced via `max-width`
- Counter-based section numbering: `counter-reset: section` + `content: "0" counter(section)`
- `font-variant-numeric: lining-nums tabular-nums` on data columns

### From SecurityScorecard
- Letter grade (A-F) as primary compliance indicator — pre-learned mental model
- Single character communicates the summary at headline scale

### From GOV.UK
- Pixel-based spacing (not rem) for component isolation
- Uni-directional margins to eliminate conflicts
- No ornamental illustration — every pixel serves information

### From FT / Datawrapper
- `font-variant-numeric: tabular-nums` for vertical digit alignment
- Annotation positioned adjacent to data, not in separate legends
- Sparklines for trends instead of full charts

## Anti-Patterns to Avoid (The Generic Death List)

1. **Inter font at every scale** — signals "default" not "designed"
2. **Purple-to-blue gradient hero** — actively signals "generated by AI"
3. **Bento grid with border-radius 12-24px** — every B2B product 2023-2025
4. **Glassmorphism** — `backdrop-filter: blur` creates accessibility failures
5. **Dark-mode-everything** — formal documents are printed white
6. **Three-word taglines** — "Build. Ship. Scale." tells nothing
7. **Generic 3D blob illustrations** — no semantic connection to product
8. **Rainbow salad color encoding** — colors without consistent semantic meaning
9. **Pie charts for compliance** — humans can't compare angles; use bars
10. **Circular progress rings for every metric** — reserve for THE most important one
11. **Animated growing bars / counting numbers** — delay comprehension
12. **Gradient status badges** — flat color + high contrast text is most legible

## The "Designed by a Human" Test

A design feels human-made when it shows **constraint that costs something:**
- A paid typeface license (Tiempos, Financier, Sohne)
- A color that breaks the "safe" palette (FT pink, Vanta dark purple)
- `font-variant-numeric: tabular-nums` on data columns
- Tufte-style stripped tables (no borders, precise alignment)
- CSS counter-based section numbering (editorial decision, not template)

## CSS Quick Reference

```css
/* Compliance ring gauge (CSS only) */
.score-ring {
  background: conic-gradient(var(--color) calc(var(--pct) * 1%), #e5e7eb 0);
  border-radius: 50%;
}

/* Tabular numerals */
.data { font-variant-numeric: lining-nums tabular-nums; text-align: right; }

/* Fluid editorial headline */
.headline { font-size: clamp(2rem, 4vw + 1rem, 5rem); letter-spacing: -0.03em; text-wrap: balance; }

/* Editorial section counter */
body { counter-reset: section; }
section { counter-increment: section; }
section::before { content: "0" counter(section); }

/* Tufte borderless table */
table { border-collapse: collapse; }
th { border-bottom: 2px solid currentColor; }
td { border: none; padding: 0.25em 0.75em; }
```
